# 04 — UX Specification

## Design Principles

1. **Everything visible** — no hidden menus, no drilling into subpages for common operations
2. **Patchbay mentality** — routing is a grid, not dropdown menus
3. **Studio hub workflow** — optimised for someone with synths, outboard, and an ADAT preamp
4. **Dark theme** — designed for studio environments
5. **Conditional display** — only show ports that exist at the current sample rate and I/O mode

## Persistent Header Bar

Always visible across all tabs. Contains:

- **Connection indicator** — green LED + device name + sample rate
- **Clock sync status** — lock LED + "Locked" / "Unlocked"
- **TALK button** — toggles talkback
- **MAIN/ALT button** — toggles speaker switching
- **DIM button** — toggles -18dB dim
- **MUTE button** — toggles master mute
- **Master volume** — horizontal slider with dB readout
- **Remote clients indicator** (when server is active) — e.g. "1 client connected"
- **SOLO indicator** — yellow "SOLO" badge with Clear button, visible only when any channel is soloed

## Footer Bar

Status line: connection state, firmware version, active speakers, dim/mute/talkback state.

## Tab Structure

Five tabs: **Overview** | **Mixer** | **Routing** | **Matrix** | **Settings**

## Stereo Pairing

Outputs are logically grouped as stereo pairs wherever they share a physical connector or function. The UI treats each pair as a single stereo channel, not two mono channels.

### Default Stereo Pairs (18i20 Gen 3)

| Pair Name | Outputs | Physical | Fader | Meter |
|-----------|---------|----------|-------|-------|
| Mon 1 (MAIN) | 1/2 | Rear panel jacks | Single linked fader | Stereo (L+R) |
| Mon 2 (ALT) | 3/4 | Rear panel jacks | Single linked fader | Stereo (L+R) |
| Line 5-6 | 5/6 | Rear panel jacks | Single linked fader | Stereo (L+R) |
| HP 1 | 7/8 | Front panel stereo jack | Single linked fader | Stereo (L+R) |
| HP 2 | 9/10 | Front panel stereo jack | Single linked fader | Stereo (L+R) |

### Behaviour
- A stereo pair shows as one channel strip with a single fader that controls both L and R simultaneously
- The VU meter shows L and R bars side by side within the strip
- Solo/mute applies to both channels in the pair
- The routing tab still shows individual L/R connections (you might route different sources to L vs R)
- In the overview, outputs are listed by pair name ("HP 1") not by individual channel ("HP 1 L", "HP 1 R")
- Future: option to unlink a stereo pair for independent L/R control (edge case, low priority)

### Input Stereo Pairing
Inputs are NOT auto-paired — in a studio hub scenario, each input likely carries a different source (different synth, different mic). Inputs always render as individual mono strips. Future: optional user-defined stereo pairing for inputs.

## Overview Tab

A read-only status dashboard. "What's going on right now."

### Front Panel LED Mirror

A horizontal strip showing all LED states from the hardware front panel. Reads the same notification data as the other views but displays it as LEDs matching the hardware layout:

- USB, LOCK, MIDI status LEDs
- 48V indicators (×2 groups)
- Per-channel INST (×2), AIR (×8), PAD (×8) LEDs
- TALK, ALT, DIM, MUTE LEDs

**No VU meters in the LED strip** — the meters are already visible in the input section below, so duplicating them here is redundant.

### Input Section

All hardware inputs displayed as **vertical channel strips in a single row**, grouped with labels below:

```
┌──────────────────────────────┬─────────┬──────────────────────────────┐
│  1   2   3   4   5   6   7  8│  L   R  │  1   2   3   4   5   6   7  8│
│  ▮   ▮   ▮   ▮   ▮   ▮   ▮  ▮│  ▮   ▮  │  ▮   ▮   ▮   ▮   ▮   ▮   ▮  ▮│
│  ▮   ▮   ▮   ▮   ▮   ▮   ▮  ▮│  ▮   ▮  │  ▮   ▮   ▮   ▮   ▮   ▮   ▮  ▮│
│  ▮   ▮   ▮   ▮   ▮   ▮   ▮  ▮│  ▮   ▮  │  ▮   ▮   ▮   ▮   ▮   ▮   ▮  ▮│
│        Analogue              │  S/PDIF │          ADAT                │
└──────────────────────────────┴─────────┴──────────────────────────────┘
```

Each channel shows:
- **Channel number** at top (1-8 for analogue, L/R for S/PDIF, 1-8 for ADAT)
- **Vertical VU meter** with peak hold
- **Route destination badge(s)** below the meter — e.g. "Mon 1 L", or "unrouted" in grey

Groups are visually separated with a subtle divider and labelled below (Analogue, S/PDIF, ADAT). Each group only appears if the device has those ports at the current sample rate.

This compact layout means analogue and ADAT inputs sit side by side in the same row — you see all 18 hardware inputs at once. The channel numbers are local to each group (1-8, not 1-18), matching how Focusrite and most musicians think about them.

DAW inputs are NOT shown on the overview — they're only relevant in the mixer and routing tabs.

### Output Section

Below the inputs. A grid showing each analogue output with:
- Horizontal level bar
- Output name with MAIN/ALT badge where applicable
- Source currently routed to it (or "none")
- Dimmed/opacity reduced if that output is currently inactive (e.g. MAIN outputs when ALT is selected)

### Status Widgets

Bottom row, small cards:
- **Speakers:** MAIN / ALT toggle with active indicator
- **Phantom Power:** 48V 1-4 / 48V 5-8 status
- **Talkback:** active status + routing target summary

## Mixer Tab

Channel strips with faders for active mixing.

### Layout

Collapsible groups arranged vertically:
- **Analogue** (default: expanded) — 8 channels + 48V controls at top of group
- **S/PDIF** (default: expanded) — 2 channels
- **ADAT** (default: expanded) — 8 channels
- **DAW** (default: collapsed) — 20 channels, compact faders

Each group header shows: group name, channel count, expand/collapse toggle.

### Channel Strip Components (per channel)

From top to bottom:
1. Channel number
2. VU meter (vertical, with peak hold) + fader side by side
3. dB readout
4. Mute (M) button — red when active
5. Solo (S) button — yellow when active (software solo: mutes all non-soloed channels)
6. Divider line
7. Hardware controls (analogue group only):
   - INST button (inputs 1-2 only)
   - PAD button (inputs 1-8)
   - AIR button (inputs 1-8)

Solo is software-only — the 18i20 has no native solo. When any channel is soloed, the header bar shows a "SOLO" indicator with a Clear button.

## Routing Tab

Full patchbay matrix of ALL port types.

### Axes

- **Rows (sources):** Analogue In, S/PDIF In, ADAT In, Mixer Out (A-L), DAW Out (1-20)
- **Columns (destinations):** Analogue Out (1-10), S/PDIF Out, ADAT Out, Mixer In (1-25), PCM In (1-20)

### Layout

- Group headers on both axes with colour coding
- Each crosspoint is a clickable button
- Active routes highlighted in purple
- One source per destination (clicking a crosspoint auto-clears the column)
- Scrollable in both directions for the full matrix
- Preset buttons: "Direct" (1:1 mapping), "Clear All"

### Colour Coding

| Group | Colour |
|-------|--------|
| Analogue | Emerald/green |
| S/PDIF | Sky/blue |
| ADAT | Amber/yellow |
| Mixer | Rose/pink |
| DAW/PCM | Violet/purple |

Consistent across all tabs.

---

## Matrix Tab

DSP mixer gain crosspoint grid.

- 25 inputs × 12 mix bus outputs (A through L)
- Click to toggle between off and 0dB
- Future: drag or scroll to set precise gain (-80dB to +6dB)
- Active crosspoints shown in green, brightness proportional to gain
- Preset buttons: Direct, Preamp, Stereo Out, Clear

---

## Settings Tab

Form layout, max-width constrained.

### Audio Settings
- Sample rate selector (44.1 – 192 kHz)
- Clock source (Internal, S/PDIF, ADAT)
- Sync status indicator (LED + text)

### Digital I/O Mode
- Radio buttons with descriptions for each mode
- Warning banner when optical I/O is disabled at high sample rates

### Monitor Configuration
- MONITOR knob assignment (which outputs it controls)
- DIM/MUTE button assignment (which outputs they affect)
- Talkback routing (which outputs receive talkback)
- Talkback mode (momentary vs latching)
- Speaker switching enable/disable

### Server (Remote Control)
- Server port (default 18120)
- Server enable/disable toggle
- Connected clients list
- Authentication PIN
- Meter update rate for remote clients

### Device
- MSD mode toggle (with warning)
- Firmware version display
- Save config to file / Load config from file
- Save config to hardware flash (with warning about flash wear)

---

## Responsive Behaviour (iPad)

When accessed from the iPad client:
- Fader hit targets enlarged (min 44px wide)
- Horizontal scrolling for mixer groups with momentum
- Landscape orientation preferred for mixer tab
- Optional "Performer mode" — restricted view showing only selected channels and a single headphone mix
- Overview and mixer tabs are the primary views; routing/matrix/settings accessible but less frequently used
